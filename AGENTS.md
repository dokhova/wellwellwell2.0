# Правила проекта WellWellWell

## Данные
- Любое пользовательское действие, меняющее данные, обязано иметь Supabase-путь через src/app/lib/api/*. Локальный стейт и localStorage — только кэш поверх базы, не источник истины.
- В отчёте по задаче всегда указывать: какая таблица, что пишется, откуда читается при перезаходе.
- Если задача требует изменения схемы БД — выдать SQL отдельным блоком в начале отчёта с пометкой «ВЫПОЛНИТЬ ДО ДЕПЛОЯ». Код фронта должен переживать отсутствие новой колонки (не ронять весь запрос).
- Счётчик и список всегда считаются из одного источника данных.

## Демо-контент (мёртвые души)
- Ограничения для демо-профилей применяются только при isDemo === true, никогда по косвенным признакам.
- Демо-профили участвуют в общих механиках (подписки, комментарии, ленты) наравне с реальными, кроме личных сообщений реальным пользователям.

## Аналитика
- posthog-js импортируется только в src/app/lib/analytics.ts. Все события — через типизированный track.
- Новое пользовательское действие = новое событие в analytics.ts (согласовать имя в задаче).

## Код
- Пути к файлам с кириллицей нормализовать через NFC.
- После каждой задачи: npm run build, ошибки сборки чинить в рамках задачи.
- Диплинки: только формат https://t.me/WellWellWell_New_bot?startapp=..., поле shareUrl из данных не использовать.
